## 🛡️ Risk Mitigation (Cross-Model Warnings)

All 3 models flagged the same failure modes. Here's the consensus defense:

| Risk | Flagged By | Mitigation |
|---|---|---|
| **"Boring B2B" anti-pattern** | GLM, Gemini | Make the Auditor Portal look like Bloomberg Terminal / enterprise software, NOT a hackathon toy. Glassmorphism + mono font + dark mode. |
| **Batch-send SDK friction** | DeepSeek | Day 1 validation: send 3 individual transfers, not a real batch. Simulate batch via sequential sends. If SDK fights, fall back to single-recipient demo. |
| **Demo "waiting for other users" problem** | GLM | Pre-load 3 mock employees. All transfers execute instantly in demo. Never wait for real users. |
| **Key management UX** | GLM | Abstract away. Employer clicks "Generate Key" → copies to clipboard → pastes in Auditor window. No key file management. |
| **"Where's the AI?" fatigue** | Gemini | DO NOT add AI unless scope allows. The compliance proof IS the technical wow. Don't dilute with a chatbot wrapper. |
| **PDF looks like a toy** | All 3 | Use `jsPDF` or `react-pdf`. Template from real IRS form aesthetics: borders, headers, watermarks, certification language. Professional → believable → fundable. |
